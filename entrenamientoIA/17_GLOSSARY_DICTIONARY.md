# 17 - Gran Diccionario y Glosario de Términos

> Basado en Documento Explicativo 15
> Términos técnicos, legales y de negocio de ProcureData.

---

## Categoría: Infraestructura y Desarrollo Web

| Término | Definición |
|---------|------------|
| **API** | Application Programming Interface. Interfaz que permite la comunicación automatizada entre ProcureData y sistemas externos como ERPs |
| **Backend** | Lógica de servidor y gestión de bases de datos que procesa las peticiones de los usuarios |
| **CORS** | Cross-Origin Resource Sharing. Protocolo de seguridad que permite o restringe el acceso a recursos desde dominios externos |
| **Deno** | Entorno de ejecución para TypeScript usado en las Edge Functions por su seguridad nativa |
| **Edge Function** | Código que se ejecuta en la periferia de la red (cerca del usuario) para minimizar la latencia |
| **Frontend** | Interfaz visual construida en React con la que interactúan los usuarios |
| **JSON-LD** | Formato de datos enlazados que proporciona contexto semántico a la información intercambiada |
| **JWT** | JSON Web Token. Estándar de seguridad para transmitir identidades verificadas entre cliente y servidor |
| **Lovable** | Ecosistema de desarrollo Full-Stack donde se aloja la interfaz y lógica de ProcureData |
| **PostgreSQL** | Motor de base de datos relacional que custodia las 28 tablas del sistema |
| **REST** | Representational State Transfer. Estilo de arquitectura para el diseño de APIs ligeras y escalables |
| **Supabase** | Plataforma de servicios de backend que integra base de datos, autenticación y funciones serverless |
| **TypeScript** | Lenguaje de programación que añade tipado estático a JavaScript para garantizar robustez |
| **UUID** | Universally Unique Identifier. Código alfanumérico único asignado a cada organización y transacción |

## Categoría: Blockchain, Web3 y Finanzas

| Término | Definición |
|---------|------------|
| **ABI** | Application Binary Interface. Diccionario técnico para que el código "hable" con los Smart Contracts |
| **Blockchain** | Registro digital compartido e inmutable que actúa como el notario de todas las operaciones |
| **Chain ID** | Identificador numérico de la red blockchain (Pontus-X: 32460) |
| **DID** | Decentralized Identifier. Identidad digital que pertenece a la empresa y no a una plataforma central |
| **ERC-20** | Estándar técnico de los tokens fungibles en redes compatibles con Ethereum, como el EUROe |
| **Escrow** | Mecanismo de custodia donde el pago se retiene hasta que los datos son validados |
| **EUROe** | Moneda digital (Stablecoin) con paridad 1:1 con el euro físico, regulada bajo MiCA |
| **Gas Fee** | Tarifa mínima pagada a la red por la computación necesaria para registrar datos en blockchain |
| **Hash** | Huella criptográfica única generada a partir de un archivo para verificar que no ha sido alterado |
| **Inmutabilidad** | Garantía de que una vez registrado un dato en la red, nadie puede borrarlo ni modificarlo |
| **MiCA** | Markets in Crypto-Assets. Reglamento de la UE que otorga el marco legal para el EUROe |
| **Pontus-X** | Infraestructura blockchain oficial de Gaia-X que sustenta la confianza de ProcureData |
| **Smart Contract** | Contrato digital que se ejecuta solo cuando se cumplen las condiciones acordadas |
| **Wallet** | Billetera digital que almacena los EUROe y las llaves privadas para firmar transacciones |
| **Zero-Knowledge Proof** | ZKP. Método para probar que una información es cierta sin revelar el contenido del dato |

## Categoría: Gobernanza de Datos e IDSA

| Término | Definición |
|---------|------------|
| **Asset** | Activo. Cualquier conjunto de datos o servicio disponible para intercambio en el Marketplace |
| **Clearing House** | Servicio que facilita la liquidación de acuerdos y el registro de cumplimiento |
| **Conector EDC** | Eclipse Dataspace Connector. El "enrutador" oficial para el intercambio de datos soberano |
| **Constraint** | Restricción. Regla en un contrato ODRL que limita el uso (ej: "solo para fines de auditoría") |
| **Consumer** | Consumidor. Rol de la empresa que solicita y utiliza los datos compartidos |
| **Data Holder** | Entidad que custodia físicamente el dato y lo entrega bajo instrucciones del propietario |
| **Data Space** | Espacio de Datos. Ecosistema federado donde los datos fluyen bajo normas de gobernanza comunes |
| **DSSC** | Data Spaces Support Center. Centro europeo que define las reglas para los espacios de datos |
| **Gaia-X** | Iniciativa europea para crear una nube soberana, segura y transparente |
| **IDS** | International Data Spaces. Estándar de arquitectura para el intercambio de datos confiable |
| **ODRL** | Open Digital Rights Language. Lenguaje estándar para escribir políticas de uso y licencias digitales |
| **Policy** | Política. El conjunto de permisos y prohibiciones que rigen un intercambio de datos |
| **Provider** | Proveedor. Rol de la empresa que ofrece sus datos manteniendo el control sobre ellos |
| **Soberanía del Dato** | El derecho absoluto de una empresa a decidir cómo se usa su información compartida |
| **Trust Framework** | El conjunto de reglas técnicas y legales que garantizan la confianza entre participantes |

## Categoría: Compras, Sostenibilidad e IA

| Término | Definición |
|---------|------------|
| **ARIA** | El asistente inteligente de ProcureData entrenado con 10,000+ referencias de conocimiento |
| **Benchmarking** | Comparativa del desempeño propio frente a los líderes del sector de forma anónima |
| **CSRD** | Corporate Sustainability Reporting Directive. Ley europea que obliga a reportar impacto ambiental |
| **Datos Sintéticos** | Información creada por IA que imita la realidad para pruebas sin exponer datos privados |
| **Embedding** | Conversión de texto a vectores numéricos para que ARIA entienda el significado de las preguntas |
| **ESG** | Environmental, Social, Governance. Los tres pilares de la inversión y gestión responsable |
| **Green Partner** | Badge de excelencia otorgado a proveedores sostenibles verificados |
| **Health Score** | Calificación de la organización basada en la integridad y actualidad de su información |
| **Homologación** | Proceso de validación legal y técnica de un proveedor antes de su contratación |
| **KYB** | Know Your Business. Verificación de la legitimidad legal de una empresa en el registro |
| **Lead Time** | Tiempo de espera desde el pedido hasta la entrega física de un bien o servicio |
| **nxm** | Fórmula que representa la ineficiencia de las validaciones redundantes en compras tradicionales |
| **RAG** | Retrieval-Augmented Generation. Técnica que usa ARIA para consultar documentos antes de responder |
| **ROI** | Return on Investment. Cálculo del ahorro económico generado por el uso de ProcureData |
| **Scope 3** | Emisiones indirectas producidas en la cadena de suministro, rastreadas por la plataforma |
| **Z-Score** | Indicador estadístico que predice la probabilidad de riesgo financiero de un proveedor |

## Reglas de Respuesta para ARIA

### Triggers de Glosario
- "qué significa" / "qué es" / "definición de" → Buscar en glosario y definir
- término técnico desconocido → Ofrecer explicación simplificada

### Formato de Respuesta
Cuando el usuario pregunte por un término:
1. Proporcionar definición concisa
2. Indicar la categoría del término
3. Relacionar con contexto de uso en ProcureData

### Ejemplo
> **Usuario**: ¿Qué es ODRL?
> **ARIA**: ODRL (Open Digital Rights Language) es el lenguaje estándar para escribir políticas de uso y licencias digitales. En ProcureData, todos los contratos de intercambio de datos se expresan en formato ODRL para que las máquinas puedan ejecutarlos automáticamente.
